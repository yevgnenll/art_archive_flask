image_archive를 api로 설계

우리 API는 다음 method를 사용합니다

  • GET: 서버에서 데이터를 찾아옵니다.
  • POST: 서버에 데이터를 입력합니다.
  • PUT: 서버의 데이터를 갱신합니다.
  • DELETE: 서버의 데이터를 삭제합니다.

image_archive의 Endpoint

Action HTTP method URI
유저의 이미지 list GET /api/images/?page=page&count=count&user_id=user_id
이미지 업로드 POST /api/images/

image_archive의 Resources

Resource URI
이미지 list /api/images/
이미지 upload /api/images/

1. 유저의 이미지 list 보여주기

  • URL

/api/images/?user_id=user_id

  • Method

    GET

  • URL Param

required:
user_id=[Integer] 유저의 ID

  1. user_id가 있는 경우
    업로드한 유저ID: user_id=[String]

    ex.) /api/images/?user_id=101

  2. user_id가 없는 경우
    요청 실패: 아래의 ERROR Response 참조

    • SUCCESS Response

    • code: 200
      content: 1페이지에 10개 이하의 데이터 전송

         ({
    	image_url: 이미지 주소[URL],
    	title: 이미지 제목[String]
        thumbnail_url: thumbnail 이미지 주소[URL],
    	})

    • ERROR Response

    • code: 404
      content: 

       { error: "Data doesn't exist" }

    • code: 500
      content: 

       { error: "Internal Server Error" }

    • Sample Code

  $.ajax({
    url: "/api/images/?user_id=101”,
    dataType: "json",
    type : "GET",
    success : function(result) {
      console.log(result);
    }
  });

2. 이미지 업로드하기 api

이미지를 업로드 합니다.

  • URL

    /api/images/

  • Method

    POST

  • URL Param

    • required:
      user_id=[Integer]
      title=[String]
      image_data=[Byte]

    • discription:
      user_id로 사용자 정보가 연결됩니다.
      작품에 대한 고유 ID는 자동으로 부여됩니다
      multipart 로 업로드를 해야 image가 업로드 됩니다

  • SUCCESS Response

    • code: 201
      content: 
       { result: "Created"}

  • ERROR Response

    • code: 400
      content: 

       { error: "Bad Request" }

    • code: 500
      content: 

       { error: "Internal Server Error" }

  • Sample Code

  $.ajax({
    url: "/api/images/",
    dataType: "json",
    type : "POST",
    data : {
        "title": “이미지제목",
        “user_id": 업로드하는 사용자의 id,
        "image_data”: “멀티파트 업로드되는 이미지 바이트 데이터”
        }
    success : function(result) {
      console.log(result);
    }
  });
